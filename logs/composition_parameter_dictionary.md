## Identification

| Column | Type | Description |
|--------|------|-------------|
| `filename` | string | 출력 WAV 파일명. `Script_{N}_{LLLL}.wav` 형식. N=스크립트 번호, LLLL=4자리 라인번호 |

## Scoring Dimensions (D1-D7)

모든 점수는 [0.0, 1.0] 범위. **1.0 = 최고 품질**.

| Column | Dimension | Full Name | Description |
|--------|-----------|-----------|-------------|
| `S_unprompted` | D1 | Unprompted Whisper Similarity | GT 텍스트를 initial_prompt로 주지 **않은** 상태에서 Whisper가 transcribe한 결과와 GT의 CER 기반 유사도. 오디오에 실제로 무엇이 담겨있는지에 대한 편향 없는 측정. 현재는 eval_results에 unprompted 결과가 없으면 기존 similarity를 fallback으로 사용 |
| `S_prompted` | - | Prompted Whisper Similarity | GT 텍스트를 initial_prompt로 준 상태의 유사도. S_gap 계산을 위한 보조값. 독립 점수 차원은 아님 |
| `S_gap` | D2 | Prompted-Unprompted Gap | `1.0 - max(0, S_prompted - S_unprompted)`. 두 유사도의 격차. 격차가 크면 GT prompt가 실제 오디오 결함(truncation 등)을 보상하고 있다는 신호. **0.85 이하 = truncation 의심**, **0.70 이하 = 강한 truncation** |
| `S_snr` | D3 | Signal-to-Noise Ratio Score | 음성 구간(voice onset~offset) 대비 무음 구간의 SNR을 [0,1]로 매핑. `min(1.0, SNR_dB / 40.0)`. 단일 화자 스튜디오 녹음에서 일관되게 높아야 함 |
| `S_duration` | D4 | AST Duration Anomaly Score | 세션 레벨 AST(Average Spoken Time, 글자/초(chars/s)) 분포 대비 z-score 기반 점수. `1.0 - min(1.0, abs(z) / 3.0)`. z=0(평균)이면 1.0, z=±3 이상이면 0.0. 같은 녹음 세션 내에서 발화속도가 유의미하게 벗어나는 segment를 탐지. 한국어는 교착어 특성상 단어 수보다 글자 수가 발화시간과 더 잘 대응됨 |
| `S_confidence` | D6 | Whisper Self-Reported Confidence | Whisper가 segment별로 반환하는 `avg_logprob`을 sigmoid 매핑. `no_speech_prob > 0.3`이면 30% 감점, `compression_ratio > 2.0`이면 20% 감점. Whisper 자체가 불확실하다고 판단한 구간을 반영 |
| `S_boundary` | D7 | Boundary Integrity Score | R2(경계 무음) + R6(envelope 적합) 이진 판정을 [0,1] 연속 점수로 확장. 경계 무음 margin(얼마나 -40dB보다 더 낮은지) 40% + envelope 목표 달성률(pre-attack/tail 비율) 60% 가중 합산 |
| `S_stability` | D5 | Transcription Stability | Whisper를 temperature 0.0/0.4/0.8으로 3회 실행 시 pairwise CER의 일관성. **현재 미구현** (PENDING segment에 대해 on-demand 실행 예정). 빈 값 = 미산출 |
| `S_composite` | - | Composite Score | D1,D2,D3,D4,D6,D7의 **기하평균**. 단일 차원이 낮으면 전체를 강하게 끌어내림 (산술평균보다 엄격). verdict 결정의 핵심 입력값 |

## AST Statistics

| Column | Type | Description |
|--------|------|-------------|
| `ast_z` | float | 해당 segment의 AST z-score. 양수 = baseline보다 빠름(fast), 음수 = baseline보다 느림(slow). **±2.0 이상이면 주의**, **±3.0 이상이면 강한 이상치** |
| `ast_p` | float | AST z-score의 양측 검정 p-value. **p < 0.05이면 통계적으로 유의미한 이상치** → `HUMAN_REVIEW_NEEDED` 플래그 부여. p = 1.0은 baseline과 완벽히 일치하거나 미산출 |

## Flags

| Column | Type | Description |
|--------|------|-------------|
| `flags` | string (pipe-delimited) | 세그먼트에 부여된 플래그 목록. `\|`로 구분. 주요 값들은 아래 표 참조 |

### Flag Values

| Flag | Meaning |
|------|---------|
| `HUMAN_REVIEW_NEEDED` | AST p < 0.05로 발화시간이 통계적으로 유의미하게 비정상. 인간 청취 검토 권장 |
| `AST_anomaly_fast` | 세션 평균보다 유의미하게 빠른 발화 — truncation 또는 merge 에러 의심 |
| `AST_anomaly_slow` | 세션 평균보다 유의미하게 느린 발화 — 과도한 silence 포함 또는 잘못된 alignment |
| `z=N.NN,p=N.NNNN,AST=N.NNNvsN.NNN,session=XXX` | AST 이상치 상세정보. z-score, p-value, 실제 AST, baseline AST, 세션 ID |
| `formal_ending_truncation_risk` | 한국어 형식 어미(-습니다 등)로 끝나는 문장인데 발화시간이 baseline보다 유의미하게 빠름 (z > 1.5). 끝음절 잘림 가능성 |
| `unprompted_not_available` | evaluate_dataset.py에서 unprompted score가 아직 산출되지 않아 기존 similarity를 fallback으로 사용 |
| `ast_not_computed` | AST baseline이 없어서 S_duration을 기본값(0.5)으로 처리 |
| `hard_reject:S_unprompted` | S_unprompted < 0.50으로 즉시 REJECT |
| `hard_reject:S_gap` | S_gap < 0.50으로 즉시 REJECT |
| `hard_reject:S_snr` | S_snr < 0.30으로 즉시 REJECT |

## Thresholds (Calibrated)

| Parameter | Value | Method |
|-----------|-------|--------|
| `tau_accept` | ~0.748 | Known-good 분포의 5th percentile (bootstrap 1000회 median) |
| `tau_reject` | 0.65 | Default (known-bad 샘플 부족 시 고정값) |
| Hard reject: S_unprompted | < 0.50 | 절대 하한 |
| Hard reject: S_gap | < 0.50 | 절대 하한 |
| Hard reject: S_snr | < 0.30 | 절대 하한 |
| AST significance | p < 0.05 | 양측 z-test |
| AST z-score cap | ±3.0 | S_duration 점수 매핑 상한 |

## Composition Decision Flow

```
1. Hard reject gates (어느 하나라도 해당 시 즉시 REJECT):
   S_unprompted < 0.50  OR  S_gap < 0.50  OR  S_snr < 0.30

2. Composite score = geometric_mean(S_unprompted, S_gap, S_snr, S_duration, S_confidence, S_boundary)

3. If S_stability available:
   composite *= stability_gate(S_stability)

4. Decision:
   composite >= tau_accept (0.748)  →  ACCEPT
   composite >= tau_reject (0.650)  →  PENDING
   composite <  tau_reject          →  REJECT
```
